Tokyo Last Train API

東京終電検索 API ドキュメント — Base URL: http://localhost:8080

1. Station Search / 駅検索 2. Last Train Route / 終電経路検索

1. Station Search / 駅検索

GET /api/v1/stations/search

Search stations by name. Supports English and Japanese queries with partial matching.
駅名で検索します。英語・日本語の部分一致に対応。最大20件を返します。

Request Parameters

NameTypeRequiredDescription
query string required Station name query (e.g., shibuya, 渋谷)

Example Request

GET /api/v1/stations/search?query=shibuya

Response

{
  "stations": [
    {
      "stationId": "odpt.Station:TokyoMetro.Ginza.Shibuya",
      "nameJa": "渋谷",
      "nameEn": "Shibuya",
      "railway": "odpt.Railway:TokyoMetro.Ginza",
      "railwayNameJa": "銀座線",
      "railwayNameEn": "Ginza Line",
      "operator": "odpt.Operator:TokyoMetro",
      "latitude": 35.6580,
      "longitude": 139.7016
    }
  ]
}

Response Fields

FieldTypeDescription
stationsarrayList of matching stations (max 20)
stations[].stationIdstringUnique ODPT station identifier. Use this for the last-train query.
stations[].nameJastringStation name in Japanese
stations[].nameEnstringStation name in English
stations[].railwaystringODPT railway identifier
stations[].railwayNameJastringRailway/line name in Japanese
stations[].railwayNameEnstringRailway/line name in English
stations[].operatorstringODPT operator identifier
stations[].latitudenumberGeographic latitude
stations[].longitudenumberGeographic longitude

2. Last Train Route / 終電経路検索

GET /api/v1/last-train

Find the last available train routes between two stations. Results are filtered to departures after the current time. Uses the RAPTOR algorithm for optimal pathfinding with up to 3 transfers.
2つの駅間の終電を検索します。現在時刻以降の出発のみ表示。最大3回乗換まで対応。

Request Parameters

NameTypeRequiredDescription
from string required Departure station ID (e.g., odpt.Station:TokyoMetro.Ginza.Shibuya)
to string required Arrival station ID

Example Request

GET /api/v1/last-train?from=odpt.Station:TokyoMetro.Ginza.Shibuya&to=odpt.Station:JR-East.Yamanote.Tokyo

Response

{
  "fromStation": "odpt.Station:TokyoMetro.Ginza.Shibuya",
  "toStation": "odpt.Station:JR-East.Yamanote.Tokyo",
  "calendarType": "Weekday",
  "routes": [
    {
      "departureTime": "23:45",
      "arrivalTime": "0:12",
      "railway": "odpt.Railway:TokyoMetro.Ginza",
      "railwayNameJa": "銀座線",
      "railwayNameEn": "Ginza Line",
      "railDirection": "odpt.RailDirection:TokyoMetro.Asakusa",
      "trainType": "Local",
      "destinationNameJa": "浅草",
      "destinationNameEn": "Asakusa",
      "transfers": [
        {
          "stationNameJa": "赤坂見附",
          "stationNameEn": "Akasaka-Mitsuke",
          "fromRailway": "odpt.Railway:TokyoMetro.Ginza",
          "toRailway": "odpt.Railway:TokyoMetro.Marunouchi",
          "departureTime": "23:50"
        }
      ],
      "totalFare": 320
    }
  ]
}

Response Fields

FieldTypeDescription
fromStationstringDeparture station ID (echoed)
toStationstringArrival station ID (echoed)
calendarTypestringWeekday, Saturday, or Holiday
routesarrayAvailable last train routes, sorted by latest departure first

Route Object

FieldTypeDescription
departureTimestringDeparture time in HH:MM format (24h). Times after midnight shown as 0:XX
arrivalTimestringArrival time in HH:MM format
railwaystringFirst segment's railway ID
railwayNameJastringRailway name in Japanese
railwayNameEnstringRailway name in English
railDirectionstringRail direction identifier (nullable)
trainTypestringTrain type: Local, Rapid, Express, etc. (nullable)
destinationNameJastringFinal destination in Japanese (nullable)
destinationNameEnstringFinal destination in English (nullable)
transfersarrayTransfer details. Empty array if direct route
totalFarenumberTotal fare in JPY (IC card price). 0 if fare data unavailable

Transfer Object

FieldTypeDescription
stationNameJastringTransfer station name in Japanese
stationNameEnstringTransfer station name in English
fromRailwaystringRailway ID before transfer
toRailwaystringRailway ID after transfer
departureTimestringDeparture time from transfer station
Note: The calendarType is automatically determined by the server based on the current date in the Asia/Tokyo timezone. Specific holidays registered in the ODPT calendar data take priority over day-of-week detection.

Error Responses

StatusCause
400Missing required parameters (query, from, or to)
